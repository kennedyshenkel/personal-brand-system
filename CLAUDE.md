# Personal Brand Content System

All writing produced in this project must conform to the voice defined in `voice-style-reference.md` and the quality floor defined in `anti-slop-rules.md`. Read both before producing any content.

## Getting Started

If this is your first time using this system, follow this path:

1. **Run `/setup`** to build your brand profile and discover your voice. Takes 20-40 minutes. You can complete it in multiple sessions.
2. **Review your generated files** in the vault. Check `brand-profile.md` and `voice-style-reference.md`. Edit anything that feels off.
3. **Try `/personal-writer`** to write your first piece. See how the voice feels.
4. **Run `/voice-check`** on the output to audit it against your style guide.
5. **Run `/refine`** after writing 3-5 pieces to sharpen your voice profile with real output data.

### Skill Progression

| Stage | Skills Available | Gate |
|-------|-----------------|------|
| Setup | `/setup`, `/grill-me` | None |
| Core | `/personal-writer`, `/voice-check`, `/brainstorm`, `/research`, `/repurpose`, `/content-calendar` | Identity + Voice + Pillars completed |
| Refinement | `/refine` | Minimum viable profile complete |

Content skills will not produce output until your minimum viable profile is complete (identity, voice, and at least 3 content pillars). Run `/setup` first.

## Critical Rules

These apply to ALL work in this project, not just skills:

1. **Anti-slop rules override everything.** `anti-slop-rules.md` is the highest-priority reference. When in doubt, check it.
2. **Never use em dashes or en dashes.** Use spaced periods (". . ."), commas, colons, parentheses, or two sentences.
3. **Never use unconditionally banned phrases.** "delve," "In today's...," "game-changer," "let's dive in," "In conclusion," "navigate the complexities," "it's important to note that" — see full list in anti-slop-rules.md Section 1.
4. **If it could come from any brand unnoticed, it's wrong.** Every piece must carry this specific voice.
5. **All output goes inside `vault/`.** No exceptions.
6. **Research brief feedback requires re-research.** If the user reviews a research brief and provides feedback that warrants new or additional sourcing, re-invoke `/research` in supplemental mode before passing to `/personal-writer`. Never send an expanded or redirected argument to the writer without sourcing the new claims first.
7. **LinkedIn content must follow the LinkedIn playbook.** All skills producing, planning, auditing, or repurposing LinkedIn content must load `linkedin-playbook-reference.md`. The full playbook (`linkedin-playbook.md`) is the human-readable source of truth. If they conflict, the full playbook governs.
8. **Check setup progress before running content skills.** If `setup-progress.md` shows the minimum viable profile is incomplete, redirect the user to `/setup`.

## Output Conventions

**File naming**: `YYYY-MM-DD-slug.md` (e.g., `2026-05-11-ai-marketing-panic.md`)

**YAML frontmatter** on all drafts:
```yaml
---
register: professional | personal
form: long-form | short-form
platform: linkedin | facebook | instagram | essays | blog | [other]
type: article | post | caption | essay | commentary
status: draft | final
created: YYYY-MM-DD
topic: [brief description]
---
```

**Draft-to-published workflow**: When a piece is approved, update `status: final` and move from `drafts/` to the parent platform folder.

## Adding New Skills

1. Create a new directory in `.claude/skills/` named after the skill
2. Add a `SKILL.md` file inside with YAML frontmatter (`description`, `user-invocable: true`)
3. Place any supporting files alongside `SKILL.md` in the same directory
4. Register it in the Available Skills table above
5. Follow the same output conventions and file structure
